# 账号运营 SOP（养号 / 冷启动）— 发布前 checklist

> 本文件是「内容生产流水线」之外的**纯平台运营底子**，不进 9 段技能接口。目标：让产出的好内容在小红书 / 公众号上**发得出去、长得起来粉**。涨粉 = 好内容 × 平台适配 × 稳定节奏，前两项靠流水线，这一项是平台适配的运营动作。

## 0. 先定变现（一切的起点）
- 用 `account.md` 的变现路径（小红书引流 → 私域/知识星球 → 选题库订阅）倒推内容方向。
- 每条内容发布前自问：**它把人往漏斗的哪一环推？**（引流型 / 信任型 / 转化型）
- ⚠️ 内容做起来才发现赛道难变现 = 最贵弯路。先定赚钱方式，再做账号。

## 1. 养号期（注册后 3~5 天，只浏览不发文）
- 每天刷 20~30 分钟**同赛道**内容，点赞、收藏、少量评论（像活人）。
- 建立"活人轨迹"：算法才认你是真实用户，后续发的笔记才有基础曝光。
- 研究手机（看内容、拆爆款）与发文手机分开；发文机建议二手 iOS（流量更稳）。

## 2. 账号门面（发第一篇前定好）
- **账号名**：简洁体现定位 + 价值（参考同赛道爆款命名，别用纯英文名/乱码）。
- **简介**：一句话说清"你是谁 + 能给读者什么"，不硬留联系方式（用 `publish_check` 扫硬引流）。
- **头像/背景**：真人感 or 统一风格，别用默认。

## 3. 发布节奏（稳定 > 爆发性）
- 新号前 2 周：**每周 3~4 篇**，固定时段（如早 8 / 晚 8），让算法识别更新规律。
- 用流水线 `topics` 产出的周排期（哪天发/什么类型/套哪条公式）执行。
- 单篇发布后 1 小时内自己点赞+回复前几条评论，撬动初始互动。

## 4. 发布前 double-check（流水线出口）
- [ ] `ziliaoku-gate` 的 `publish_check` 已过（`pass=true`，无绝对化/收益数字/硬引流/AI味残留）
- [ ] 标题套用了 `titles_pool` 公式且符合目标平台风格（小红书短钩子+emoji；公众号长悬念+干货感）
- [ ] 封面符合 `image-styles` 已验证风格
- [ ] `cta` 是提问式，非"关注我/加微信"

## 5. 数据回收（喂回流水线）
- 每周把发布数据（小眼睛/点赞/收藏/评论/涨粉）填进 `ziliaoku-review`，反哺 `formulas.md` / `image-styles.md` 权重。
- 哪类结构/封面涨粉好 → 下次 `collect` 的「拆平台爆款」重点拆同类。

## 红线（与 account.md 一致，发布前必守）
- 不提具体收益数字
- 不用绝对化用语
- 不硬引流（不直接留微信/主页放外链）
- AI 生成必须人工过一遍，尤其标题和前 3 句
