# 出稿前简报模板（brief）

> 移植自 launch-pack 起号手册「笔记简报模板 / 文章简报模板」（2026-07-10）。
> 用法：出稿前先填这一页对齐，再让 `ziliaoku-draft` 出正文，减少返工。
> 按 `platform` 选对应模板；`task` 取自 `ziliaoku-topics` 选题的 `task` 字段。

## 小红书笔记简报（platform=xhs）

| 模块 | 填写内容 |
| --- | --- |
| 笔记任务 | 搜索 / 信任 / 互动 / 证明 / 转化 / 留存 |
| 目标读者 | 谁会在刷到时停下来？ |
| 关键词 | 1 个主关键词 + 2 到 4 个相关词 |
| 封面承诺 | 用户点进来的可见理由 |
| 标题组 | 8-12 个，标出推荐（复用 draft「标题多候选」） |
| 开头钩子 | 前 1-2 行：痛点/结果/冲突/故事 |
| 正文结构 | 3-6 段，每段一个观点+一个细节/证据 |
| 证据 | 用户自有图片/案例/截图/过程/经历 |
| 人味细节 | 一个真实场景/限制/错误/观点 |
| 行动引导 | 一个与任务匹配的平台内动作 |
| 标签 | 品类/产品/场景/问题/人群标签 |
| 风险检查 | 声明/版权/隐私/医疗金融法律风险/平台规则 |

## 公众号文章简报（platform=公众号）

| 模块 | 内容 |
| --- | --- |
| 文章任务 | 搜索 / 信任 / 故事 / 情绪 / 证明 / 转化 / 留存 |
| 目标读者 | 谁最应该点开这篇 |
| 读者状态 | 痛点/好奇/误解/恐惧/渴望/反对意见 |
| 关键词 | 1 主 + 2-4 相关 |
| 标题组 | 8-12 个，标推荐 |
| 开头钩子 | 前 1-3 行：痛点/冲突/结果/场景 |
| 正文结构 | 3-6 节，每节一观点一证明/细节 |
| 故事节点 | 冲突 → 原因 → 信息 → 情绪转折 |
| 证明材料 | 用户自有案例/截图/经验/来源/过程 |
| 风格 | 口语化、短句，必要时用"你/我" |
| 行动引导 | 与任务匹配的一个站内动作 |
| 风险检查 | 宣传承诺/版权/隐私/医疗金融法律风险/平台规则 |

## 朋友脑改写（草稿写完后自检）
- 把端着的表达改成能说出口的话；拆短句子。
- 加具体场景、例子、利害关系；删空泛总结段。
- 一篇只保留一个核心承诺。
