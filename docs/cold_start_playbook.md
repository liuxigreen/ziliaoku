# 30 天冷启动阶段剧本

> 来源：对比 chenjin-cmd/agent-skills-launch-pack_ 的「30 天账号冷启动日历」后，提炼适配本账号（AI Agent 实操工作流博主）的版本。
> 定位：本剧本**只指导节奏与阶段目标**，不替代流水线（collect→gate→extract→cluster→signal→topics→draft→image→review）。选题与成稿仍走技能。

## 核心原则
- 不买粉、不互关、不刷数据；只用内容本身养号。
- 小红书是主战场（对标 chenjin 包的 xiaohongshu-account-launch-expert），公众号/视频号/X 为侧翼。
- 养号期（前 14 天）发布 ≤3 篇/周，不挂任何外链、不提私域、不写收益数字（守 `account.md` 红线）。
- 每周日跑 `ziliaoku-review` 复盘，数据驱动调向（不拍脑袋）。

## 五阶段剧本

### 阶段 1 · 定位（Day 1-3）
- **目标**：账号第一眼的信任基建立住，避免"全领域 AI 博主"式空泛。
- **动作**：
  - 填 `account.md` 的「定位句模板」+「门面系统」（昵称/简介/开场白/内容支柱/关键词宇宙）。
  - 确定 3 根内容支柱轮换比：① 工具/Git 拆解 ② 爆款写法拆解 ③ 自身复盘/观点。
  - 准备好门面（头像/昵称/简介/置顶）再发第一篇。
- **产出**：`account.md` 门面系统填完；首篇选题锁定（引流型，低门槛高传播）。

### 阶段 2 · 对标（Day 4-7）
- **目标**：找到 5-10 个同 niche 已起号样本，拆他们的标题/封面/结构，喂本系统。
- **动作**：
  - 用 `opencli reddit search` / 小红书发现页，找 5-10 个"AI 工具实操"对标账号。
  - 把高赞帖喂 `ziliaoku-decode` 拆写法，沉淀进 `data/winning_cases.jsonl`。
  - 跑 `ziliaoku-topics` 产出首周选题库（≥5 个资源型/获得感选题）。
- **产出**：`winning_cases` ≥5 条；首周 `output/topics_{week}.md` 含周排期。

### 阶段 3 · 首轮（Day 8-14，养号期）
- **目标**：稳定更新建立算法信任，验证内容方向。
- **动作**：
  - 按周排期发 3 篇（引流型为主，穿插 1 篇自身复盘建人设）。
  - 每篇成稿前跑 `publish_check`（C1 合规自检）+ 标题 8-12 候选选优（draft 多候选规则）。
  - 发布后记录真实数据（小眼睛/赞/藏/评/涨粉）到 `reviews/{week}.md` 模板。
- **产出**：3 篇发布 + 首周复盘草稿；识别 1 个相对爆点方向。

### 阶段 4 · 放大（Day 15-23）
- **目标**：把首轮验证的相对爆点方向加倍投入，拉涨粉。
- **动作**：
  - 用「9 条内容实验测款方法论」(`content_experiment.md`) 在同一方向做角度矩阵，找最佳角度。
  - 发布提速到 3-4 篇/周；转化型内容开始种草（软钩子 cta，不提价格）。
  - 封面模式从 `image-styles.md` 选"验证有效"的那类。
- **产出**：周更 3-4 篇；涨粉曲线抬头；封面/标题公式权重在 `formulas.md` 更新。

### 阶段 5 · 复盘（Day 24-30）
- **目标**：固化可复用方法论，决定下月方向。
- **动作**：
  - 跑完整 `ziliaoku-review`，用「失败环节分层诊断」定位最差条卡在漏斗哪层、给治法。
  - 若某入口 `discard` 率 >60% 连续两周 → 调上游初筛词（不是调严质检）。
  - 输出月度方向文档（哪类内容留、哪类砍）。
- **产出**：月度复盘 + 下月选题侧重；门面系统按需微调（连续 2 周同类差才动）。

## 与流水线映射
| 剧本阶段 | 调用的技能 |
|---|---|
| 定位 | account.md（手写） |
| 对标 | opencli reddit + ziliaoku-decode |
| 首轮/放大 | ziliaoku-topics → ziliaoku-draft(+publish_check) → ziliaoku-image → 半自动发布 |
| 复盘 | ziliaoku-review |

> 本剧本是「运营节奏层」，技能是「生产层」。两层不混：剧本不写具体文案，技能不拍发布节奏。
