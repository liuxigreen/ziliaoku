# 内容创作方法论（爆款底层框架）

> 来源：@yanliudreamer《内容起号后，如何让它真正被放大、爆起来》（用户 2026-07-08 粘贴的总结版）。
> 定位：**写作工艺参考层**，供 `ziliaoku-draft`(Prompt-E) 调用——教我们"怎么写出更容易爆的帖"，**不是选题源、不是 AI 工具、不进 collect/gate**。
> 与我们账号的关系：我们是"用 AI Agent 做内容工作流的实操博主"，这篇是**跨平台通用的爆款工艺**，可直接套到我们的 AI 工具 niche。

---

## 一、爆款核心公式
**爆款 = 点击率 × 停留时长 × 互动率**
（前提：在小流量测试中，数据明显优于平均值——爆款不是运气，是测出来的）

| 因子 | 决定什么 | 受什么影响 |
|---|---|---|
| 点击率 | 能不能进场 | 封面 + 标题，用户刷到的第一秒 |
| 停留时长 | 有没有价值 | 是否一开讲重点、有无废话、结构清晰 |
| 互动率 | 进入更大流量池 | 收藏 > 评论 > 转发（**收藏权重最高**，是"价值认可"强信号） |

## 二、万能内容结构（4 段模板）
1. **钩子**（前 3 秒 / 第一行）—— 制造好奇或痛点
   - 例："90% 的人起号失败，都错在这一步" / "我用错方法浪费了 3 个月"
2. **真实背景**（建立信任）—— 一句话来历
   - 例："我从 0 开始做小红书" / "这是我试错后的总结"
3. **核心信息**（控制在 3 点以内）—— 清晰易吸收，不信息轰炸
4. **总结 + 引导互动** —— 鼓励评论提问，拉互动率（尤其收藏）

## 三、5 种最容易爆的类型（按我们账号的套法）
1. **情绪共鸣型（最涨粉）**：展示脆弱 + 真实经历 → 建立强认同
   - 我们可套："我做 AI 内容最崩溃的一天" / "被限流后才懂的事"
2. **结果展示型（最建信任）**：先亮结果再讲过程，结果越真实越有说服力
   - 我们可套："我用 Agent 把一周内容生产压到 2 天" / "半自动发帖跑通后省了多少时间"
3. **认知反差型（点赞最多）**：打破原有认知，带来"原来如此"
   - 我们可套："越努力越没流量？你以为的爆款逻辑其实是错的" / "全自动发帖反而废号"
4. **信息差干货型（收藏最多）**：直接给实用价值，让人"收藏了以后一定用得上"
   - 我们可套："我常用的 AI Agent 工作流" / "10 个每天在用的 AI 工具" / "一套效率翻倍的 Prompt"
5. **身份认同型（最易被转发）**：帮用户表达态度/价值观，转发=表达"这是我的观点"
   - 我们可套："AI 时代作品集比简历重要" / "未来每人都该有自己的内容流水线"

## 四、写前 3 问自检（回答不清 = 大概率不爆）
1. 这条内容解决了**谁的问题**？
2. 用户**为什么要点进来**？
3. 看完能**带走什么**？

## 五、底层 5 种需求（所有爆款最终满足其一）
❤️ 情感共鸣 ｜ 📈 见证成长 ｜ 💡 获得新知 ｜ 💻 学习提升 ｜ 🚀 身份表达

## 六、起号后期提醒
- 出现小爆款后，算法会把你**归类进某个内容池**（AI / 职场 / 成长…）。
- **顺着爆款方向深耕，别突然乱发**——否则流量会掉。
- 我们账号的动作：哪类（信息差干货 / 认知反差）先小爆，就往那类集中产出。

## 七、与我们 draft(Prompt-E) 的对接点
- **structure_ref 默认套"4 段模板"**：钩子必须第一句进痛点/好奇；核心信息 ≤3 点。
- **content_angle 从 5 种类型选**（我们账号优先：信息差干货型 + 认知反差型 + 结果展示型；情绪共鸣型慎用，避免过度卖惨）。
- **成稿前用"写前 3 问"过一遍**，答不清就重写。
- **互动钩子放第 4 段**（总结 + 引导），目标是拉收藏 / 评论（非转发）。
- **仍守红线**：不提收益数字 / 不绝对化 / 不硬引流 / 去 AI 味 / 不画饼（见 `ziliaoku-draft` SKILL.md）。
- **数据观**：爆款是"小流量测试里优于均值"，不是一次性赌大爆——对应我们 review 阶段的"小爆深耕、不乱发"纪律。
