# 对标账号拆解表（融合 launch-pack 小红书 §9 + 公众号 §4）

> 我们 `collect` 目前只抓**单篇爆文**，缺**账号级对标**。本表补「账号级模式库」——用于 `topics` 参考「形式 / 钩子模式」，而非只看单篇。

## 好对标的选择标准（两平台共识）

- 受众相似、体量可参考：粉丝 < 1 万但互动明显高于体量最佳；中腰部号里出现重复爆文结构也可。
- 近 3 个月有出圈 / 明显高互动；主页反复出现相似封面、标题或内容结构。
- 评论区暴露痛点、异议、用户原话和购买意向。
- 不追求大号；不批量矩阵、不滥用账号、不人为制造增长。

## 对标记录表（合并版，带 platform 列）

| 字段 | 记录内容 | 适用 |
| --- | --- | --- |
| 账号 | 名称 / 定位 / 赛道 / 读者 / 可见粉丝信号 | 全平台 |
| 文章/笔记 | 链接 / 标题 / 日期 / 主题 / 形式 / 互动 | 全平台 |
| 钩子 | 用户为什么会点进来 | 全平台 |
| 关键词/标题模式 | 搜索词；或标题数字 / 对比 / 热词 / 疑问 / 对话 | xhs / wechat |
| 封面/结构 | 版式承诺 / 视觉证明；或故事 / 清单 / 教程 / 观点 | xhs / wechat |
| 情绪触发 | 好奇 / 身份 / 焦虑 / 松弛 / 实用收益 | wechat |
| 证明 | 图片 / 截图 / 数据 / 过程 / 身份 / 反馈 | 全平台 |
| 行动引导 | 评论 / 收藏 / 关注 / 提问 / 咨询（须平台内合规） | 全平台 |
| 评论挖掘 | 用户原话、真实问题、异议 | 全平台 |
| 重建方向 | 如何用自有证据 / 表达改写结构（**不搬原文**） | 全平台 |

## 使用方式

- 在 `collect` / `gate` 阶段，除抓单篇爆文外，定期（对标期 4-7 天）填 5-10 个账号进本表。
- 存储建议：`data/benchmarks/{platform}.jsonl`（待建），供 `topics` 参考「形式 / 钩子模式库」。
- 标签 / 合集只能当研究线索，不承诺带来推荐流量；标签须真实、相关、稳定。
- 对标只学结构，不搬原文；用我们自己的素材和证明重新写。

## 存储格式（落地 data/benchmarks/）

每个平台一个 jsonl 文件：`data/benchmarks/{platform}.jsonl`，每行一条对标记录（字段对应上表）：

```json
{
  "account": "账号名 / 定位 / 赛道 / 读者",
  "post": "链接 | 标题 | 日期 | 形式 | 互动",
  "hook": "用户为什么点进来",
  "title_pattern": "数字/对比/疑问…（xhs/wechat）",
  "cover_or_structure": "版式承诺 / 故事清单…",
  "proof": "截图/数据/过程",
  "cta": "平台内行动引导",
  "comment_mining": "用户原话/异议",
  "rebuild": "用自有素材改写的方向"
}
```

`collect` / `gate` 阶段填 5-10 个账号进对应平台文件，供 `topics` 参考"形式/钩子模式库"。
