读完本栏目，你能在「加渠道 / 加产品 / 加 Agent / 加团队」之前拿走 5 件产物：
...
- 一张放大准备度评分卡，先确认瓶颈在哪（单价 / 复购 / 时间 / 状态 / 现金流）再加码
- 一份可交接的运营 SOP（接单 / 沟通 / 交付 / 复盘），把重复动作写成“另一个人也能做”的清单
- 一份渠道扩展实验设计（Fiverr / Upwork / 私域 / 领英 / 推特），用 30 天小实验验证新入口质量
- 一份 Agent 护栏（自动化 ≥ 2 周稳定 / 失败成本可控 / 非客户专属环节才能交给 Agent）
- 一份团队与资产系统（案例集 / 客户邮件 / 评论资产 / SOP 库），让一次接单变成长期资产
...
- 已经稳定接单 ≥ 90 天，月入 $3000+ 但每天工时 > 12h 的「精力到瓶颈」型
- 重复性工作占比 > 50%，想招人但 SOP 没写的「外包犹豫者」
- 单平台沉淀完成，想加 1-2 个新渠道但怕分散精力的「多渠道犹豫者」
- 接单累到想做课程 / SaaS / 模板包套餐的「产品化犹豫者」
...
- 每天 > 12h 还不知道瓶颈在哪：先去 01 评分卡定位「单价 / 复购 / 时间」哪一格最红
- 想招人但 SOP 没写：去 02 把最重复的 3 项工作写成 SOP，再决定要不要招
- 想从主平台铺第二平台：去 03 设计 30 天小实验，先验证新入口的真实询单质量
- 想用 Agent 替代客服 / 数据：去 04 把“客户专属环节”圈出来，只让 Agent 跑非专属
- 接完单就没下文 / 没复购：去 05 设计案例 + 邮件 + 评论 + 转介绍循环
...
1. 用 01 评分卡给“单价 / 复购 / 售后 / 时间 / 状态”5 维标红黄绿，最红的那 1 维就是本季主战场
2. 列出过去 30 天最重复的 3 项工作，挑 1 项写 SOP（02）或交给 Agent（04）做 2 周试运行
3. 给历史客户发一封「最近 3 个月有没有新需求 / 谁可能用得上」的邮件，启动 05 的复购 + 转介绍
