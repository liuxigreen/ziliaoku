本文汇集了 Anthropic 公司 Claude Code 团队的工作流程技巧。这些实践涵盖并行执行、规划、自动化、验证和自定义——团队每天用来更快交付代码的模式。每个人的设置都不同，所以请尝试找到适合你的方法。

### /batch 用于大型迁移
...
`/batch` 命令会询问您有关迁移的信息，然后将工作分配给所需数量的工作树代理——数十个、数百个或更多。每个代理独立工作，测试自己的更改，并独立创建 PR。

## 构建前规划
...
按 **Shift+Tab** 进入规划模式。投入精力完善计划，以便 Claude 可以一次性完成实现。典型流程是：进入规划模式 → 完善计划 → 切换到自动接受编辑 → Claude 执行。
...
运行 /effort 选择您的努力级别。可用级别为 **low**（更少令牌，更快）、**medium**、**high**（更多令牌，更高智能）、**xhigh**、**max** 和 **auto**（Claude 按请求选择）。Team、Enterprise 和直接 API 访问的默认值为 **high**，其他计划的默认值为 **medium**。Claude Code 团队对所有事情都使用 high。对于复杂的编码和代理工作，切换到 xhigh 以获得比 high 更深入的推理，而不需要 max 的完整令牌成本。对于困难的调试或架构决策，切换到 max，您希望 Claude 尽可能长时间地进行推理。Max 会更快地消耗使用限额，因此请按会话激活它。

## 验证——第 1 个提示
...
在进行更改后，将 `/simplify` 附加到任何提示。它运行并行代理来审查更改的代码，以确保代码重用、质量、效率和 `CLAUDE.md` 合规性——一次完成所有检查。

## 权限和安全
...
运行 `/permissions` 来预允许常见的安全命令并将其检入你的团队的 `.claude/settings.json`。这是 **跳过权限的推荐替代方案** — 你会收到更少的提示，同时保持可审计的允许列表。**完全支持通配符语法** — 尝试 `"Bash(bun run \*)"` 或 `"Edit(/docs/\*\*)"`。
...
自动模式让 Claude 代表你做出权限决定。分类器在每个操作运行前评估它 — 安全操作会自动批准，风险操作仍会被标记。使用 `claude --enable-auto-mode` 启用它；启用后，**Shift+Tab** 在会话期间循环 `default → acceptEdits → plan → auto`。没有该标志，循环是 `default → acceptEdits → plan`。

## 移动和远程控制
...
要求 Claude 使用 `bq` CLI 即时拉取和分析指标——在您的代码库中检查 BigQuery 技能。Claude Code 团队的看法："我个人在 6 个多月内没有写过一行 SQL。"这适用于任何具有 CLI、MCP 或 API 的数据库。
...
启用 Slack MCP，将错误线程粘贴到 Claude 中，然后说**"修复"**——零上下文切换。或者说**"去修复失败的 CI 测试"**而不进行微观管理。将 Claude 指向**docker logs** 以排查分布式系统——它在这方面的能力令人惊讶。
...
插件捆绑 LSP（适用于每种主要语言）、MCP、技能、代理和自定义钩子。从官方 Anthropic 插件市场安装，或为您的组织建立内部市场——然后将市场参考检查到 `settings.json` 中，以便为每个开发人员自动添加。运行 `/plugin` 以开始。
...
运行 `/config` 以设置浅色/深色模式，运行 /terminal-setup 以在 IDE 终端、Warp 或 Alacritty 中启用**Shift+Enter** 换行（Apple Terminal 不支持 `/terminal-setup`）。对于 Vim 键绑定，打开 `/config` → 编辑器模式。该团队推荐**Ghostty** 以实现同步渲染和 24 位颜色。
...
`/color` 更改提示输入颜色——当您打开 3-5 个会话并需要一目了然地区分它们时很有用。
...
### 语音输入
...
