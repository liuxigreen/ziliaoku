## ​文生图工作原理
...
整个文生图过程可以理解为一个反向扩散过程。我们下载的 v1-5-pruned-emaonly-fp16.safetensors是一个预训练模型，它可以从纯高斯噪声中生成目标图像。我们只需要输入提示词，它就可以通过对随机噪声进行去噪来生成目标图像。 我们需要了解两个概念
...
1. 潜空间 (Latent Space)： 潜空间是扩散模型中的一种抽象数据表示方法。将图像从像素空间转换为潜空间可以减少存储空间，并使训练扩散模型和降低去噪复杂度变得更容易。这就像建筑师使用蓝图（潜空间）进行设计，而不是直接在建筑物（像素空间）上进行设计，这在保留结构特征的同时显著降低了修改成本
2. 像素空间 (Pixel Space)： 像素空间是图像的存储空间，也就是我们看到的最终图像，用于存储像素值。
...
## ​ComfyUI 文生图工作流节点详解
...
### ​A. Load Checkpoint 节点
...
此节点通常用于加载图像生成模型。一个`checkpoint`通常包含三个组件：`MODEL (UNet)`、`CLIP`和`VAE`
...
- `MODEL (UNet)`：UNet 模型，负责在扩散过程中进行噪声预测和图像生成
- `CLIP`：文本编码器，将我们的文本提示词转换为模型可以理解的向量，因为模型无法直接理解文本提示词
- `VAE`：变分自编码器，负责在像素空间和潜空间之间转换图像，因为扩散模型在潜空间中工作，而我们的图像处于像素空间
...
### ​B. Empty Latent Image 节点
...
定义一个输出到 KSampler 节点的潜空间。Empty Latent Image 节点构建一个纯噪声潜空间 您可以将其功能理解为定义画布大小，这决定了我们最终生成图像的尺寸
...
### ​C. CLIP Text Encoder 节点
...
用于对提示词进行编码，这些提示词是您对图像的要求
...
- 连接到 KSampler 节点的`Positive`条件输入代表正向提示词（您希望在图像中出现的元素）
- 连接到 KSampler 节点的`Negative`条件输入代表反向提示词（您不希望在图像中出现的元素）
...
提示词由来自`Load Checkpoint`节点的`CLIP`组件编码为语义向量，并作为条件输出到 KSampler 节点
...
### ​D. KSampler 节点
...
KSampler 是整个工作流的核心，整个去噪过程都在这里发生，最终输出潜空间图像 以下是 KSampler 节点参数的说明
...
| 参数名称 | 描述 | 功能 |
| --- | --- | --- |
| 模型 (model) | 用于去噪的扩散模型 | 决定生成图像的风格和质量 |
| positive (正向提示词) | 正向提示词条件编码 | 引导生成过程包含指定元素 |
| negative (负向提示词) | 反向提示词条件编码 | 抑制不想要的内容 |
| latent_image | 待去噪的潜空间图像 | 作为噪声初始化的输入载体 |
| seed (种子) | 用于噪声生成的随机种子 | 控制生成的随机性 |
| control_after_generate | 生成后的种子控制模式 | 决定批量生成中种子的变化模式 |
| steps (步数) | 去噪迭代步数 | 步数越多，细节越精细，但处理时间越长 |
| cfg | 无分类器指导标度 (Classifier-free guidance scale) | 控制提示词的约束强度（过高会导致过拟合） |
| sampler_name | 采样算法名称 | 决定去噪路径的数学方法 |
| scheduler (调度器) | 调度器类型 | 控制噪声衰减率和步长分配 |
| denoise (去噪强度) | 去噪强度系数 | 控制添加到潜空间的噪声强度，0.0 保留原始输入特征，1.0 是完全噪声 |
...
在 KSampler 节点中，潜空间使用`seed`作为初始化参数来构建随机噪声，并将语义向量`Positive`和`Negative`作为条件输入到扩散模型中 然后，根据`steps`参数指定的去噪步数执行去噪。每个去噪步骤都使用`denoise`参数指定的去噪强度系数对潜空间进行去噪，并生成新的潜空间图像
...
### ​E. VAE Decode 节点
...
将 KSampler 输出的潜空间图像转换为像素空间图像
...
### ​F. Save Image 节点
...
预览并保存从潜空间解码的图像到本地的`ComfyUI/output`文件夹
...
