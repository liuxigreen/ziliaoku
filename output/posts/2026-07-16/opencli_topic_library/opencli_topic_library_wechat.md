# 别再手动刷热点了：我用 0 元搭了个「大佬动态」选题库

做内容这行，最折磨人的从来不是写，是「今天写什么」。

说个真实场景：某天你刷到一条特别想写的 AI 工具动态，兴冲冲打开文档，结果一搜——已经 30 篇写过了，而且都发在过去的 48 小时里。你犹豫半天，要么硬着头皮写个二手解读，要么干脆放弃。这种「想写却写不进去」的挫败，我经历过太多次。

后来我琢磨明白一件事：问题不在你写得慢，在于你靠「刷到啥算啥」做选题。等你从热榜、推荐流里看到某个话题，它早就被算法筛选、排序、喂到你面前了——这时候同行早写完了。你拿到的，是二手货。

所以我干了一件事：用一套完全免费的工具，把十几个我认可的 AI 大佬的 X（推特）和 Reddit 动态，实时拉成一个「候选选题库」，每天早上扫一眼，挑能落地的写。整套东西 0 元，不接任何付费 API。

## 为什么必须有一手信源

先把概念说清楚。「一手」和「二手」差的是一个时间差：

- **二手信源**：热榜、推荐流、别人的整理帖。特点是已经被平台筛过、排过序。你看到的不是全貌，是平台「想让你看到」的那部分。
- **一手信源**：你主动关注的、具体的人和社区，他们当天真实在聊什么。这里有滞后红利——一个大佬刚发的一个 workflow 技巧，可能要两天后才慢慢爬上热榜。

做内容要赚的，就是这个时间差。在话题还没发酵时，你先自己 replicate 一遍，再用大白话讲给读者，这才有「我先发现了」的价值感。

举个我常碰到的例子：某个开源 AI 工具发布了新版本，作者第一时间在 X 上发了演示。热榜上可能两天后才出现相关讨论。如果你盯的是大佬的 X，你第一天就能写；如果你只刷热榜，你就晚了两天——而那两天，已经够十个号把读者喂饱了。

## 方案：opencli 免费直采

核心工具叫 opencli，本质是一个走「本机浏览器登录态」的命令行采集器。

这里有个关键认知，也是它能免费还不封号的根本原因：**它不模拟爬虫去跟平台对抗，而是复用你浏览器里已经登录好的账号态**。你平时怎么刷 X，它就怎么拉数据——平台在请求层面看不出区别，自然不会拦你。

对比一下市面上的付费方案，你会更清楚它的位置：

- **付费 MCP（如 agentkey、firecrawl）**：按月或按量计费，能穿透长文、能抓更刁钻的页面，适合有商业采集需求的团队。
- **opencli 免费直采**：0 元，覆盖 X / Reddit / YouTube / GitHub 等主流平台，对个人创作者完全够用。

我的判断很直接：**个人选题库这种轻量需求，根本不值得花钱**。把付费额度留给真正卡住你的场景（比如某篇锁区的长文非要穿透），才是划算的。日常盯大佬动态这件事，免费方案就能漂亮地解决。

## 我的四步落地

说人话，我是这么搭起来的，你照着也能复现：

**第一步，列一份「值得跟」的名单。**
不是越多越好，是越准越好。我选了两类人：一类是做 AI 工具、会发第一手 release 的；一类是爱分享自己 workflow 的实操派。控制在 10-15 个，维护成本低，也不会信息过载。这份名单我会每季度review一次，把不再产出干货的人换掉。

**第二步，用 opencli 拉他们的动态。**
X 上拉指定账号的 timeline，Reddit 上拉我常看的几个 subreddit 的新帖。一行命令就能拿到结构化结果，不用自己写爬虫、不用处理登录鉴权。

**第三步，关键词过滤。**
不是所有动态都值得写。我设了几条规则，比如只要出现 agent / workflow / 开源 / 评测 这类词才进候选池。这样每天递给我的不是 200 条噪音，而是 10 条能用的。过滤规则你可以按自己的领域调，做设计的就盯 design / 做开发的就盯 code。

**第四步，早上扫一眼，挑能落地的。**
这一步必须人来。AI 能帮你「收集 + 过滤」，但「这个我能不能 replicate、读者会不会关心」得你自己判断。我一般花 5 分钟过一遍候选池，定下当天 1-2 个选题，剩下的标个「观察」，过两天看热度再决定。

## 我的实际节奏

搭好之后，我的每周产出基本是这样转的：

周一扫一眼候选池，定本周主线；周二到周四每天从池子里挑一个能落地的写；周五回头看哪些「观察」项升温了，决定下周追不追。整个过程，选题决策从「刷到啥算啥」变成了「我主动从一手信源里挑」。

最直观的变化：我不再有「今天没东西写」的焦虑了。池子里永远躺着十几个候选，我只需要选，不需要从零想。

## 几个避坑点

免费的东西，最怕「用力过猛」把自己作死。这几条是我踩过或见过别人踩的：

- **节奏慢一点。** 别一秒钟发十几个请求，把请求间隔拉开，像人一样慢慢刷。今天被限流了就明天再来，不硬刚。我们自己的抓取脚本，每个源之间默认隔开 3 秒再加随机抖动，就是这个道理。
- **别当爬虫用。** 它的优势恰恰是「像人」，你非要拿它做大规模抓取，优势就没了，还容易触发封号。定位成「个人助理」，别定位成「抓取机器」。
- **隐私别乱碰。** 只拉公开动态，不碰私信、不碰别人不想公开的东西。这是底线，也是它长期能跑的前提。
- **别依赖单一平台。** X 和 Reddit 各盯一部分，某个平台抽风了你还有另一个。鸡蛋分开放，选题雷达才稳。

## 它和另一条自动化是搭档

如果你也搭了那种「定时抓 RSS 聚合源、AI 打分排序」的每日信息简报，这两件事是互补的：

- RSS 简报管「广」——每天扫一遍全网英文信源，让你知道大方向、大趋势。
- 大佬动态库管「深」——盯住具体几个人，第一时间抓到他们的新动作、新工具。

一个看面，一个看点。合起来，就是我的完整选题雷达：面上有方向，点上有抓手。
