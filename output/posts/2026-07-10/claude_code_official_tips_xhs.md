# 正文

Anthropic 把自己团队天天用 Claude Code 的内部技巧公开了。

我扒完整篇，挑出**能直接抄、当天就能用**的 5 条，附上我自己的踩坑理解 👇

**1️⃣ 官方亲口说：最有用的一招是「让 AI 检查自己」**

原文原话——「如果你只学一个技巧，就学这个：验证。」

做法：任何一句指令后面，加一个 `/simplify`。它会另开并行小号，把刚写的代码按「复用、质量、效率、规范」重审一遍。

我的理解：AI 写代码最大的坑不是不会写，是它写完自己不看。加这一步，等于让它交作业前先自己检查。我实测返工少一半。

**2️⃣ 别一个窗口死磕，开 3-5 个并行**

命令：`claude --worktree`（每个会话在独立的 git 工作树里跑，互不干扰）。

我的理解：以前我一个窗口等它跑完再下一个，纯浪费。现在一个改 bug、一个写测试、一个查文档，同时转。前提是任务之间不打架。

**3️⃣ 先规划，再动手——按 Shift+Tab**

进规划模式，把方案磨清楚，再切自动执行。官方原话是「投入精力完善计划，让它一次做对」。

我的理解：90% 的返工是因为一上来就让它写。多花 2 分钟把要干啥说清楚，比事后改 5 遍快。

**4️⃣ 它犯过的错，写进 CLAUDE.md**

在仓库根目录建个 `CLAUDE.md`，提交进 git。**只要 AI 做错一次，就把这条记进去**，它下次就不会再犯。

我的理解：这个最反直觉但最值。相当于给 AI 建了本「错题本」，越用越顺手，不用每次重新教。

**5️⃣ 每天做超过一次的事，就存成技能**

官方规矩：一件事一天做超过一次，就把它写成 `SKILL.md` 存起来，团队共享。

我的理解：这条我最有感——我自己就在把重复的活儿一个个存成技能，攒着攒着，重复劳动肉眼可见地少了。
