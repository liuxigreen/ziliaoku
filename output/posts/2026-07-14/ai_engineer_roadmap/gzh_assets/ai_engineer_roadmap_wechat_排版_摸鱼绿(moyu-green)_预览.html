普通人能抄什么

最近刷到个老外 @sairahul1 发的「6个月成为 AI 工程师路线图」

19.7 万浏览、2400+ 赞，我没当收藏帖划走，逐月拆给你看

一张火遍全网的图，19.7 万浏览、2400+ 赞。但火不等于能落地。我没划走，而是逐月拆了一遍——哪些普通人真能抄，哪些看着漂亮其实劝退。今天把拆完的结论原样给你。

01

PART


第 1 个月：地基月，别急着冲大模型

MONTH 1 · FOUNDATION

路线图第 1 个月，要你先搞顺的不是 AI，而是编程地基。这一步最容易被跳过，但也最值钱。


Python 真写顺

不是会 print，是能写脚本处理真实数据


Git + GitHub

版本管理、协作、看别人代码


命令行 / 终端基础


JSON、API、HTTP、异步基础


基础 SQL


pandas 处理数据


虚拟环境、包管理、错误处理


FastAPI 或 Flask 起个接口

✦ 我的理解

我见过太多人第 1 天就 pip install langchain 搞 agent，结果连 JSON 都 parse 不明白，一报错就卡死。地基不牢，上面盖什么都是危房。

宁可能慢，别假会——碎片时间就把一个月拉长到两个月

02

PART


第 2 个月：正式碰 LLM，重点在会指挥不在会调

MONTH 2 · PROMPTING

这月才是 AI 核心，但他特意强调：不是背 API，是搞懂这几个机制：


提示词基础（prompting fundamentals）


system 指令 vs user 指令的区别


结构化输出 / JSON schema


function / tool calling（让模型调你的函数）


流式响应（streaming）


对话状态管理


成本 / 延迟 / token 基础


失败处理


提示注入意识（prompt injection awareness）

← 这点是真良心，务必刻进脑子

✦ 我的理解

最后这条我单独拎出来讲，因为它和我们上一篇「用 AI 读代码被反杀」是同一个坑——你以为在指挥 AI，其实是别人在代码里藏了句话指挥 AI。

别把来路不明的内容直接喂给 AI 当指令

这是第 2 个月最该刻进脑子的一课，比多背两个 API 有用

03

PART


第 3-4 个月：RAG + Agent，学什么时候不该用

MONTH 3-4 · RAG & AGENT

进入 RAG 与 Agent 阶段，要点密集：


embeddings（把文字变成可计算的向量）


chunking（怎么切分文档）


向量数据库


元数据过滤


reranking（重排序）


检索质量调优


减少幻觉


引用与溯源


agent 循环、工具选择、状态管理、重试


什么时候不该用 agent（这点比怎么用重要）

✦ 我的理解

很多人学到这就开始无脑堆 agent，啥任务都套个循环。路线图专门写了 when NOT to use agents——这才是真懂。

80% 的场景，一个好提示词 + 一次调用就够

用 agent 反而慢还贵，别被 agent 很酷带偏

04

PART


第 5-6 个月：部署 + 选方向，从会做到能交付

MONTH 5-6 · SHIP & PICK

最后两个月，从会做走向能交付：


FastAPI 生产模式


Docker


后台任务、队列


鉴权 + API key 安全


日志、可观测性


提示词 / 版本管理


评估看板


成本监控、速率限制、缓存

然后三个方向选一深挖：

方向1

AI 产品工程师 · 想快速进创业公司，偏 LLM 应用、RAG、agent、部署、产品 UX

方向2

应用型 ML / LLM 工程师 · 微调、何时微调 vs 提示、评估、推理优化、开源模型、训练管线

方向3

AI 自动化工程师 · 工作流编排、业务流程自动化、多工具系统、CRM、文档、邮件、客服、运营场景

✦ 我的理解

方向 3 其实最贴近打工人用 AI 替自己打工——你不需要进大厂，把本职的重复活（报表、客服初筛、文档整理）用工作流自动化掉，就是实打实的产出。

半年从运营转成了 AI 产品

关键不是他多聪明，是每个月都拿真实项目练，不是收藏吃灰

WRAP-UP

这张图值不值？值。但它最大的陷阱是：看着清晰，容易让人以为收藏 = 学会。

普通人能抄的只有一件事——把它当 checklist，每个月逼自己交付一个真实小项目，而不是又存进以后看的收藏夹。
