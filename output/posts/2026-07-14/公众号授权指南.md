# 公众号自动监控授权指南（wewrite-stats）

> 目标：接上后，`wewrite stats --days 7` 能直接拉公众号阅读/分享/点赞数据，和 opencli 小红书监控一样形成双平台数据飞轮。
> 当前状态：未授权，公众号数据只能手动看后台。

---

## 一、要什么凭证

wewrite-stats 调的是**微信公众号「数据统计」接口**，需要公众号的：

- **AppID**（开发者 ID）
- **AppSecret**（开发者密码）

对应 `config.yaml` 里的 `wechat.appid` / `wechat.secret`，或环境变量 `WECHAT_APPID` / `WECHAT_SECRET`（二选一，任一齐全即可）。

> ⚠️ 注意：微信「数据统计」接口**只有已微信认证的服务号/订阅号**才开放（未认证订阅号多数无此权限）。如果拿不到 AppSecret 或拉数报错 "api unauthorized"，就是账号类型限制，只能继续手动看后台。

---

## 二、去哪拿 AppID / AppSecret

1. 浏览器登录 **微信公众平台** https://mp.weixin.qq.com
2. 左侧菜单 → **设置与开发 → 公众号设置 → 开发者ID（AppID）** ← 这里复制 AppID
3. 同页 → **开发者密码（AppSecret）** → 点「重置」，扫码后显示一次（**只显示这一次，务必立刻保存**）
4. 若提示「IP 白名单」，把本机出口 IP 加到 **设置与开发 → 基本配置 → IP白名单**（否则 token 获取会被拒）

---

## 三、填到哪

方式 A（推荐，写文件）：

```bash
# 复制模板
cp ~/.wewrite/config.example.yaml ~/.wewrite/config.yaml
# 编辑 ~/.wewrite/config.yaml，把这两行改成你的：
wechat:
  appid: "wx_你的_appid"
  secret: "你的_appsecret"
```

方式 B（环境变量，不落盘更安全）：

```bash
# 临时（当前 shell 生效）
export WECHAT_APPID="wx_你的_appid"
export WECHAT_SECRET="你的_appsecret"
# 或写进 ~/.bashrc / 系统环境变量持久化
```

> 📁 `~/.wewrite/` 即 `$WEWRITE_HOME`，是 wewrite 的用户态目录（不在仓库里）。若 `WEWRITE_HOME` 未设，默认 `~/.wewrite`。

---

## 四、验证能不能拉数

```bash
# 先跑依赖诊断（确认凭证被识别）
wewrite diagnose

# 能拉数后：
wewrite stats --days 7
```

成功的话会输出最近 7 天文章阅读数据，并回填 `~/.wewrite/history.yaml`。失败会明确报 "missing appid/secret" 或微信接口错误——把报错发我，我帮你看是凭证问题还是账号类型限制。

## 五、和我这边流水线的关系

- **小红书监控**：已通（opencli `creator-*` 命令，免费直读，无需任何凭证）。
- **公众号监控**：就差上面这一步授权。接上后，双平台都能在发布后自动拉数据复盘，数据飞轮就完整了。
- **不自动发布、不自动回复**：授权只用于「读数据」，绝不用于自动发文或自动评论（主号红线）。

## 六、需要你做的事（我没法替你做）

1. 去微信公众平台拿 AppID + 重置 AppSecret（Secret 只显示一次）。
2. 把这两个值告诉我（或自己填进 `~/.wewrite/config.yaml` / 设环境变量）。
3. 我跑 `wewrite diagnose` 验证，通过就接好了。

> 如果你不想把 Secret 明文给我，最安全的方式是你自己填进 `~/.wewrite/config.yaml`（文件在本机你的 home 目录，我不会去读它），然后只告诉我"已填好，你跑 diagnose 验证"即可。
