做自媒体的朋友，有没有过这种时刻——

选题想好了，脚本卡半天，封面懒得做，剪出来又嫌丑，最后拖了一周啥也没发。

我最近挖到一个刚更新（7月4号）的开源项目，作者直接把「做视频」拆成了一串 AI Agent 流水线：从选题、写稿、做封面、搞 SEO、合成、发布到看数据，按天自动跑，人基本不用管。

说清楚：它接的是 YouTube。但我觉得它那套「工作流思路」对所有做视频的人都很有启发，所以先拆给你看。

## 它到底是什么

项目叫 **YouTube Automation Agent**，目前 1.5k 星。
核心就一句话：7 个各管一环节的 AI Agent，串成一条流水线，天天自动转。

这 7 个 Agent 分别是：

- **选题 Agent**：扒趋势、定方向、排内容日历
- **脚本 Agent**：写带钩子、有故事线、带转化的稿子
- **封面 Agent**：出图 + 做 A/B 版本
- **SEO Agent**：标题、描述、标签一把抓
- **制作 Agent**：配音、配图、视频合成
- **发布 Agent**：上传、排期、管播放列表
- **数据 Agent**：盯表现，把结论喂回选题

最关键是最后那步——**数据反哺选题，形成闭环**。不是「生成完就完事」，而是越跑越知道下一条该做啥。

> 这一套「多 Agent 各管一段 + 数据回流」的结构，其实就是内容工作流自动化的标准解法，换个项目也能套。

## 门槛和成本（讲点实在的）

部署要会一点点命令行：clone、npm install、填 .env。
硬性前提就三个：Node 18+、装个 FFmpeg、一个免费的 YouTube Data API。

AI 这边它接几乎所有 OpenAI 兼容的服务：OpenAI、Gemini、Kimi、MiMo、GLM，或者用一个 OpenRouter 钥匙通吃 300+ 模型。

比较贴心的一点是：**连 AI Key 都不用也能跑**——没 Key 的时候它自动退回内置模板兜底，流水线不会断。
想省钱的，Gemini 有免费额度，Kimi 比 GPT 便宜一大截。作者自己在文档里写的成本参考是「每条视频几美分到一毛多美元」量级（用 OpenAI 的话），属于能跑着玩的水平。

## 作者亲口说的坑（写在 v2.1 更新日志里）

这项目挺让我放心的一点：作者不藏着掖着，把踩过的坑直接摊在更新日志里：

- 一开始启动就崩，缺个 `sharp` 依赖；
- 定时任务一跑就报错，因为少建了一张表；
- 发布时竟然传了「占位视频」上去……v2.1 才修好，改成「只传真视频，占位直接拒绝」；
- 模板里以前会编数据（「90% 的人…」），现在专门清掉了。

你看，连这种「假数据」都特意拿掉——做内容的人最怕这个。

## 适合谁 / 不适合谁

**适合**：想做视频、但被剪辑和日更绑住的人；想把「内容生产」本身也自动化的人。
**不适合**：完全不想碰命令行的小白（它要你会 npm 起来）；以及只做图文、不做视频的同学。
