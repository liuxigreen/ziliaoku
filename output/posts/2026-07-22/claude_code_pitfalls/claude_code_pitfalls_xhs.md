# 我用Claude Code踩的3个坑

用 CC 半年，折腾过不少项目。这 3 个坑我替你踩了，能避就避。

① 当黑盒甩需求
丢一句"做个XX"就等结果。它跑偏你才发现，返工比写还累。
→ 先给方向+约束，让它先列计划再动手。

② 不写开发文档
上下文一丢，CC 立刻"失忆"，同一个错犯三遍。
→ 留一份 dev docs，关键决策记下来，它才不跑偏。

③ 全自动一把梭
一口气让它改全站，报错雪崩，半夜收不了尾。
→ 拆小步+用 hooks 卡错误，每步留检查点。
