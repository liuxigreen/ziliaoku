===正文开始===

刷到那期「32个技巧，16分钟从小白到量产」之前，我一直觉得自己 Claude Code 用得挺溜。

结果对着清单一条条看，最该开的开关、最该养成的习惯，我一个都没碰。那一刻真的挺尴尬的——原来我用了半年，一直在裸奔。😅

这篇不堆 32 招，只讲我踩完坑才懂、而且你今天就能用上的几个。按「地基 → 上下文 → 协作 → 规模化」来说。

【一、地基：两招吃透，已经赢过大多数人】

▶ 进项目先打 /init

别每次都重新给 AI 讲一遍项目背景。/init 会让它扫一遍代码库，自动生成一个 CLAUDE.md——相当于给项目建了张速查卡。

以后每个新会话，它自己就知道项目长什么样，你少解释、它少跑偏。成本就几秒钟，回报是整个项目周期。

▶ 永远先 Plan Mode

这个开关我以前完全忽略。开了之后：它能读代码、能列方案，但一个字都不改。等你把方案磨清楚，再放行让它动手。

我实测过，90% 的返工都是「一上来就写」作出来的。多花两分钟说清楚要干嘛，比事后改五遍快得多。

【二、上下文：别让它「失忆」】

▶ 盯住上下文窗口

AI 不是记性差，是上下文塞满了它就开始「忘事」。对话越长，它越容易前后矛盾、重复造轮子。

该开新会话就开，重要的背景沉淀进 CLAUDE.md，而不是指望它在一个超长对话里记住所有事。

【三、协作：把它当队友，不是许愿池】

▶ 给它建一本「错题本」

这招我觉得是全场最有用的。

AI 做错一次，别只是当场骂两句改掉。把这个教训写进 CLAUDE.md 并提交。下次它自己就不会再犯同样的错。

这个文件不是一次性的说明书，是越用越厚的「肌肉记忆」。用得越久，你的 AI 越懂你的项目。

【四、规模化：什么时候才需要】

32 招里有一大半是给「已经跑起来、要放大」的人准备的——多智能体、并行任务、自定义命令这些。

新手真别急着全学。你现在连 /init 和 Plan Mode 都没吃透，学那些只会更乱。

【最后说句实在的】

技巧不是收藏量，是你在哪个卡点、用上了哪一招。

/init 和 Plan Mode 这两个先吃透，你已经甩开大多数「只会对话式让 AI 写代码」的人了。剩下的，等你在真实项目里卡住了，再回头翻对应那一招，那时候你才真的记得住。

## 整理成"适合排版"的 3 条原则（这就是你说的关键）

小红书长文编辑器的一键排版，本质是**识别你的结构**再套样式。所以文字要满足：

1. **小标题清晰**：用【】或独立成行的短句做小标题，编辑器/读者一眼能分段。
2. **段落短**：每段 2-4 行，手机上不压抑。长段一定拆开。
3. **重点句独立成行**：关键结论单独一行，方便它加粗/高亮。

只要文字是这个结构，粘进去点排版就成型——不需要任何 skill。
