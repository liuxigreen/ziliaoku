# 微信公众号采集 + 过滤层验证报告（2026-07-07）

## 一、测试目的
用户质疑「副业」赛道营业价值低（"最没营业"），要求：先搜索微信公众号内容，再实跑 gate 过滤层，看它能否把低价值/软广类内容正确判成 `discard` / `hack_only`，而不是一股脑收进库。

## 二、样本来源
通过搜狗微信搜索（`weixin.sogou.com`，WebFetch 可抓列表）取两组 query：
- **组 A**（AI 工具/开发赛道）：`AI Agent 实操 教程` → 10 篇
- **组 B**（副业赛道，过滤测试主样本）：`AI 副业 变现 实操` → 11 篇

共 **21 篇**公众号卡片，字段为：标题 + 公众号名 + 摘要 + 日期（来自搜索结果页）。

## 三、架构约束（重要，影响判定层级）
- ✅ 搜狗微信**搜索列表** WebFetch 可稳定抓取（发现层可用）。
- ❌ 点击 `weixin.sogou.com/link` 抓**全文**触发验证码（反爬），WebFetch 拿不到 `mp.weixin.qq.com` 正文。
- 因此本轮是 **「标题 + 摘要 + 公众号信号」级预筛**，等同 gate 标准里的 *"热榜标题（未抓全文的）→ 直接进 titles_pool，不走质检"* 分支。
- 完整正文级 gate 需：firecrawl（带 Key）/ Exa（需 Key）/ 微信登录态 —— 当前环境下均不可靠（firecrawl 今日限速、Exa 未配 Key）。

> 结论前置：过滤层**能生效**，但当前只能作用于「发现层信号」，正文级 gate 待抓取通道打通后补跑。

## 五、过滤层生效验证结论
- ✅ **过滤层对副业软广识别率极高**：组 B 11 篇中 8 篇 `discard`（全部命中「软广伪装干货」规则），2 篇 `hack_only`（仅因"标题公式/列表钩子"可复用，进 `titles_pool`，**不进爆文库**），**0 篇 `collect`**。
- ✅ 过滤层**能区分赛道营养**：组 A（AI 工具）有 3 篇真 `collect`（一手实操文），组 B（副业）0 篇——证明「副业最没营业」的用户直觉**被数据印证**。
- ⚠️ 当前是**预筛级**（只有标题/摘要）。完整正文级 gate 待抓取通道（firecrawl 带 Key / Exa / 微信登录态）打通后，需对 `collect`/`hack_only` 候选补跑全文 gate 复核。

## 六、副业赛道处置建议
1. **副业类公众号正文营养极低**（软广化率 80%），**不建议作为 `collect` 正文源**。
2. 但其**标题公式很能打**（"0基础零成本副业!月入2000-5000+""33个落地方案"式钩子），值得保留进 `titles_pool` 当**信号参考**——只抽公式，不收正文。
3. **建议**：副业从「六赛道正文源」降级为「标题公式信号源」；或在周复盘确认持续低 collect 率后，从正文源砍掉，聚焦 **AI 工具 / AI 创作 / 自媒体** 三主线。
