# ziliaoku-review — 周复盘（Prompt-D）

## Purpose
用真实发布数据反哺流水线：哪条好（强化其公式 / 封面）、哪条差（定位是选题 / 执行 / 门面问题）、关键词怎么调、入口质量如何。这是"先冻结接口、让缺口自己暴露"的收口环节。

## When to use
- 每周一次（建议周日），用户填入本周发布数据后。

## 输入
- 本周发布数据：标题、**选题 `topic_id`**、套用公式、小眼睛 / 点赞 / 收藏 / 评论 / 涨粉。
- `formulas.md`、`image-styles.md`（待更新权重）。
- `ziliaoku-gate` 本周统计：各入口 `collect` 率（按 `source_platform` 分组）、总体 `discard` 率。
- `keywords.md`（待增删建议）。

## 效果回填（按 topic_id 归因，2026-07-10 新增）
- 发布数据回填时，用成稿携带的 `topic_id`（来自 `ziliaoku-topics`）把"发布表现"归因回"选题"。
- 闭环见 `docs/run.md`：选题 → 成稿(`topic_id`) → 发布 → 本环节按 `topic_id` 写"哪篇选题带了多少量"，反向调 `topics` 下周权重。
- 不破冻结输出契约（仅输入要求多带一个 `topic_id`）。

## 输出契约（冻结，写入 reviews/{week}.md）
```markdown
# 周复盘 {week}

## 1. 最好1条
- 公式/结构 → formulas.md 权重 +1
- 封面风格 → image-styles.md 标注"验证有效"

## 2. 最差1条（失败环节分层诊断）
- **先定位卡在漏斗哪一层**（不可直接说"整体差"，必须落到具体层）：
  1. 曝光（小眼睛低）→ 门面 / 标签 / 发布时间问题，与内容无关
  2. 点击（曝光够但点击低）→ 标题或首图问题（两病分开治：标题改钩子、首图改封面模式）
  3. 阅读（点进秒退）→ 开头3秒没接住 / 正文结构散
  4. 收藏（读完不藏）→ 干货密度不足 / 无"以后用得上"价值
  5. 评论（不互动）→ 缺争议点 / 没抛问题引导
  6. 关注（不转粉）→ 人设不清晰 / 连续价值感弱
  7. 转化（不私信 / 不订阅）→ cta 弱 / 信任未到阈值
- **每层治法不同**（切忌"整体优化"式空话）：
  - 曝光低 → 调发布时间 / 标签 / 门面，不碰内容
  - 点击低 → 换标题公式（见 titles_pool）/ 换封面（见 image-styles）
  - 阅读低 → 改开头钩子 / 补结构
  - 收藏低 → 提干货密度 / 加"可复用清单"
  - 评论低 → 文末抛争议问 / 留讨论口子
  - 关注低 → 强化人设连续性 / 系列化
  - 转化低 → 强化信任 / 软性 cta
- 依据：必须给具体数据证据（哪层指标异常），禁止"感觉"。

## 3. keywords.md 增删建议
- ...

## 4. 账号档案微调
- （连续2周同类差才调向，单周不动）

## 5. 入口质量联动
- 各入口 collect 率（按 source_platform）：...
- 总体 discard 率：... ；若 >60% 连续两周 → 调上游初筛词，不是调严质检
```

## 6. 起号阶段对照与共鸣判断（移植自 launch-pack 起号手册，2026-07-10 新增）
> 来源：launch-pack 两平台手册第 12-13 节「起号日历」「数据诊断」。用于给"弱数据"一个合理归因，避免新号期误判。
> 属于"扩诊断结构"，不破冻结接口（第 2 节 7 层漏斗仍是主诊断，本节是补充基准）。

### 起号阶段基准（新号前 30 天，别用通用阈值压自己）
- 定位期(1-3天)：建定位句/主页/关键词/20条素材，还没发或刚发。
- 对标期(4-7天)：建模式库(20-50条对标)，首轮选题库成型。
- 首轮发布(8-14天)：测 3-5 个支柱，**早期弱数据不急于下结论**，先完成最小样本量(≥10条)再判断。
- 模式放大(15-21天)：重复有效信号，改封面/标题。
- 复盘聚焦(22-30天)：排序下月重点。
- ⚠️ 第 8-14 天的弱曝光/弱互动属正常探索期，review 时标注"探索期样本"，不计入"最差1条"的硬性归因，除非已发 ≥10 条。

### 共鸣判断（在 7 层漏斗之外补一层"情绪信号"）
- 评论说"这就是我" → 情绪共鸣强，可扩展成系列/框架。
- 收藏多但关注少 → 单篇有用，账号承诺不够清晰（治：强化主页/系列）。
- 关注涨但评论弱 → 信任在形成，但互动问题太封闭（治：加具体评论问题）。
- 多篇都弱 → 重查风口/承诺/证据，及创作者是否藏了最有辨识度的部分。

## 硬性要求
- 最好 / 最差各只取 1 条，且必须给依据，禁止"感觉"。
- 最差条必须明确落到漏斗某一层（曝光 / 点击 / 阅读 / 收藏 / 评论 / 关注 / 转化），并给该层治法，不混谈"整体差"。
- `discard` 率 > 60% 连续两周 → 结论是"调上游初筛词"，**不是**把质检改严。
- 账号档案连续 2 周同类差才微调，单周波动不动。

## 副作用（执行时同步更新）
- 好条对应的公式 → `formulas.md` 该条权重 +1。
- 好条封面风格 → `image-styles.md` 标注"验证有效"。
- 增删建议 → 经用户确认后改 `keywords.md`。

## 工具依赖
纯 LLM 分析，不需外部工具。当前环境即可运行。
