# ziliaoku-draft — 成稿（Prompt-E）

## Purpose
从选题到成品文案：把 `ziliaoku-topics` 里用户勾选的选题，写成严格按结构骨架展开、且**去AI味**的小红书正文，并产出配图简报供 `ziliaoku-image` 使用。

## When to use
- 用户从 `output/topics_{week}.md` 勾选选题后，说"写这条 / 出稿 / 成稿"。

## 平台参数（标题分平台，E 项补齐）
- 输入新增 `platform`（默认 `小红书`；可选 `公众号` / `视频号` / `X` / `通用`）。
- 标题公式按平台选型（不套一种通用式）：
  - 小红书：口语 + 数字 + 痛点/反常识（例："___ 我扒出3个坑"）
  - 公众号：深度 + 价值承诺（例："我们实测了 ___，结论是 ___"）
  - 视频号：冲突 + 悬念（例："别再用 ___ 了，试试这个"）
  - X：英文 + 技术精确（例："How we automated ___ with ___"）
- `final_title` 必须贴合所选平台的标题公式风格；`formulas.md` 现有通用公式（F1-F5）作兜底，平台特化公式后续补入该文件。
- 正文语气同步微调：小红书最口语、公众号偏论述、X 偏技术直给；去 AI 味与"不画饼"规则在所有平台同样强制。

## 内容角度参数（写作工艺层，来自 docs/content_craft.md）
- 输入可选 `content_angle`，从 5 种爆款类型选一（详见 `docs/content_craft.md` 第三节）：
  - `情绪共鸣型`（最涨粉，慎用：避免卖惨）、`结果展示型`（最建信任）、`认知反差型`（点赞最多）、`信息差干货型`（收藏最多，**我们账号首选**）、`身份认同型`（最易转发）。
- 未指定时默认 `信息差干货型`（最贴合"AI 工具实操博主"人设、且收藏权重最高）。
- 角度决定 `structure_ref` 的侧重：结果展示型先亮结果再讲过程；认知反差型开篇即打破认知；信息差干货型核心信息给可复用清单。
- `docs/content_craft.md` 是爆款底层框架的完整参考（公式 点击率×停留×互动、4 段万能模板、5 种类型、写前 3 问、底层 5 需求），成稿前必读。

## 爆款案例引用（来自 ziliaoku-decode，横切资产）
- 成稿前读 `data/winning_cases.jsonl` 近 10 条（由 `ziliaoku-decode` 拆解爆款/自家爆帖产出）。
- 若本选题的 `content_angle` 或主题，与某条 case 的 `reference_for` 匹配，优先借鉴其 `reusable_pattern` 与 `hook_type`，融入 `structure_ref`，**不套死 4 段模板**。
- 借鉴的是"写法骨架"，不是抄内容；仍守去 AI 味 / 不画饼 / 开头多样性铁律。
- `fit_niche` 为 null 的 case 仅借鉴写法、不套主题。

## 输入
- 一条选题（含 `title_candidates`、`hook_draft`、`structure_ref`、`evidence`、`fit_score`）。
- `account.md` 人设与红线。

## 输出契约（冻结，写入 output/posts/{YYYY-MM-DD}/{slug}.md 或同目录 JSON）
```json
{
  "final_title": "终稿标题（≤20字，从 candidates 中选优并打磨，保留数字和钩子词）",
  "body": "正文（≤1000字，小红书正文硬上限；建议600-900字，严格按 structure_ref 骨架展开）",
  "tags": ["5-8个话题标签，混合：1-2个大流量标签 + 3-4个精准垂类标签"],
  "cta": "结尾互动引导一句（提问式，禁止'关注我'式硬引流）",
  "image_count": 4,
  "image_briefs": [
    "图1封面：核心结论大字",
    "图2：步骤1截图示意"
  ]
}
```

## 开头/标题多样性铁律（防模板化，与去AI味同等强制）
> 目的：账号是一条**内容线**（AI Agent 实操工作流），连续性靠**人设 voice（实操/接地气/不画饼）+ 主题 niche**，绝不靠复用同一个开头/标题句式。本规则防止"工具清单类"选题天然滑向同一种"我筛了 N 个只留 M 个"的模板。

1. **读近期库**：成稿前先读 `data/draft/recent_openings.jsonl`（已发布稿的 title + 正文首句 + angle，每条一 JSON 行）。
2. **双重复制禁令**：本条 `final_title` 的**句式模板** 与 `body` 首句**结构**，均不得与库中近 5 条任意一条雷同。明确禁止连续出现「我筛了 N 个 X，只留 M 个」式标题 + 「卡住 X 的不是 A 是 B」式开头这种组合。
3. **角度轮转**：若近 5 条里「信息差干货型」已 ≥2 次，本条优先换 `认知反差型 / 结果展示型 / 身份认同型`（选题本身强约束时除外）。
4. **开法菜单（按选题选，禁止只会用一种）**：
   - 痛点共鸣式：「做 X 最累的不是 A，是 B」/「做自媒体的，有没有过这种时刻——」
   - 热点反问式：「最近是不是也刷到过那种——…」
   - 反差主张式：「看了几百条 X，烂片都一个共同点：…」
   - 个人经历式：「用 X 半年，直到看见 Y 才发现自己一直在裸奔」
   - 真实复盘式：「吹完 X 的好，得说落地。用了一个月…」
5. **收尾动作**：成稿定稿后，向 `data/draft/recent_openings.jsonl` 追加一行 `{date, title, opening, angle}`，供后续防重复检索（库只保留近 30 条即可）。

## 去AI味规则（写作时强制执行，不可省略）
- 开头 3 句禁用"在当今时代 / 随着AI发展"式套话，直接进钩子。
- 每 200 字至少一处口语化表达（"说实话"、"我踩过这个坑"、"别问我怎么知道的"）。
- 加入 1 处具体的个人经历细节（从 `account.md` 人设推导，**禁止编造具体收益数字**）。
- 短句为主，单句 ≤ 25 字，段落 ≤ 3 行。
- 禁用词：赋能、抓手、闭环、颗粒度、底层逻辑（**连用两个立即重写**）。
- **排版去机械化**：禁用 `▶ ①②③ •` 这类规整符号堆砌，改用自然段落 + 口语过渡；段落长短错落，别写成清单报告。
- **降营销黑话密度**：人设 / 工作流 / 变现 / 高转化信号 等词避免密集出现，能用大白话就不造词。
- **情绪有起伏**：加入第一人称碎碎念、真实口语瑕疵、适度不完美的表达，不全程金句、不句句总结（平台对正文也会判 AI 生成，规整+黑话是主要触发源）。

## 不画饼规则（人设硬要求，与去AI味同等强制）
- **禁写未验证的未来承诺**：不得出现"等我测了 X 再补截图 / 踩坑记录""后续跑顺了再更"这类话——写了就是画饼，读者会当真。
- **只写已发生的事实**：工具 / 平台若未亲手跑过，明确标注"扒完 README / 仅看文档、未实测"，不假装用过、不预支结论。
- **已验证的可如实陈述**：如"发帖这一步我已经用半自动跑通了（AI 写稿、人点发布）"——因为这是真实发生过的，不算画饼。
- 与 `account.md` 人设三关键词"实操、接地气、不画饼"强绑定；任何成稿发布前逐句自查本条。

## 写前 3 问自检（成稿前必过，来自 docs/content_craft.md）
动笔前先回答，答不清就重写（命中任一项不清 = 大概率不爆）：
1. 这条内容解决了**谁的问题**？
2. 用户**为什么要点进来**？
3. 看完能**带走什么**？

## 标题多候选再选一（提升点击率，与开头多样性同等强制）
> 目的：标题决定点击（漏斗第二层）。只给 1-2 个候选容易平庸；竞品起号包每条先写 8-12 标题再选 1。我们对接 `topics` 的 `title_candidates`（默认 2），成稿时**扩写到 8-12 个**再选优。

1. 成稿前基于 `content_angle` + `structure_ref` + `platform`，先生成 **8-12 个标题候选**，覆盖多种公式：
   - 数字痛点式（"我用 X 踩了 3 个坑"）
   - 反常识式（"别再 X 了，真相是 Y"）
   - 结果展示式（"X 之后，我每周省出 Y 小时"）
   - 身份认同式（"做 X 的人，都该看看这个"）
   - 好奇悬念式（"是什么让 X 突然好用？"）
2. 用 `data/draft/recent_openings.jsonl` + `titles_pool.jsonl` 自检：句式不与近 5 篇雷同（守开头/标题多样性铁律）。
3. 从中选 **1 个定稿**为 `final_title`（≤20字，首选贴合 `platform` 标题公式的那条）。
4. 其余候选保留在成稿 JSON 的 `title_candidates` 字段（供 A/B 或后续复用）。

## 结构骨架（structure_ref 默认 4 段万能模板）
`structure_ref` 未显式指定时，默认套 `docs/content_craft.md` 的 4 段模板：
1. **钩子**（第一行进痛点/好奇，禁用套话）
2. **真实背景**（一句话来历，建立信任）
3. **核心信息**（≤3 点，清晰易吸收，不信息轰炸）
4. **总结 + 引导互动**（提问式 cta，拉收藏/评论）

## 硬性要求
- `body` 必须严格按 `structure_ref` 骨架展开（默认 4 段模板），不得自由发挥丢结构。
- `cta` 必须是提问式互动，**禁止"关注我"式硬引流**（踩 `account.md` 红线）。
- **honor `monetization_role`**（选题自带，来自 `ziliaoku-topics`）：
  - 若为 `转化`：正文须是强资源型（直接给可复用模板 / 清单 / Prompt，不藏私），`cta` 用**软钩子**（如"这类清单我每周都在整理"）替代硬引流；**绝不可提订阅价格、不可直给微信 / 链接**。
  - 若为 `引流`：`cta` 偏涨粉互动（提问 / 投票 / "你踩过这坑吗"）。
  - 若为 `信任`：`cta` 偏经验交流（"你是怎么处理的？评论区聊聊"）。
  - 所有类型都守 `account.md` 红线（不提收益数字 / 不绝对化 / 不硬引流）。
- `image_briefs` 每张一信息，供 `ziliaoku-image` 逐张生成提示词。

## 工具依赖
纯 LLM 写作，不需外部工具。当前环境即可运行。成品发布前必须经用户终审（红线：自动化的是生产与排期，不是决策权）。
